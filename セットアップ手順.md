## 📖 用語集（最初に読んでおくと理解が深まります）

### 開発環境に関する用語

- **Node.js（ノードジェイエス）**: JavaScriptをサーバー側で実行できるようにするツール。このプロジェクトで必須です。
- **npm（エヌピーエム）**: Node.jsのパッケージ（ライブラリ）をインストールするためのツール。Node.jsと一緒にインストールされます。
- **Git（ギット）**: プログラムの変更履歴を管理するツール。バージョン管理に使用します。
- **ターミナル（コマンドプロンプト）**: 文字でコマンドを入力して操作する画面。Windowsでは「コマンドプロンプト」や「PowerShell」を使用します。

### プロジェクト構成に関する用語

- **フロントエンド**: ユーザーが直接見る画面（ホームページ）の部分。このプロジェクトではNext.jsを使用。
- **バックエンド**: サーバー側で動く処理（データベースへの保存など）。このプロジェクトではSupabaseを使用。
- **データベース**: 情報を保存しておく場所。このプロジェクトではPostgreSQLを使用。
- **環境変数**: プログラムの設定値を保存するファイル。パスワードやAPIキーなど、秘密にすべき情報を保存します。

## ✅ ステップ3: プロジェクトの初期化

### 3-1. フロントエンド（Next.js）のセットアップ

**目的**: ホームページを作るためのプロジェクトを作成します。

**手順**:

1. **フロントエンドフォルダに移動**
   ```bash
   cd frontend
   ```

2. **既存のプロジェクトがあるか確認**
   - `frontend`フォルダに`package.json`や`app`フォルダがある場合、既にNext.jsプロジェクトが作成されています

3. **既存プロジェクトがある場合の対処**

   **パターンA: 既存プロジェクトを使用する（推奨）**
   - 既にNext.jsプロジェクトが作成されている場合は、そのまま使用できます
   - 次のステップ（Supabase Clientのインストール）に進んでください

   **パターンB: 既存プロジェクトを削除して再作成する**
   - 既存のプロジェクトを削除したい場合のみ実行してください
   - ⚠️ **注意**: 既存のコードが削除されます
   
   ```bash
   # frontendフォルダの内容を削除（Windowsの場合）
   # エクスプローラーでfrontendフォルダを開き、以下のフォルダ・ファイルを削除:
   # - .next/
   # - app/
   # - node_modules/
   # - eslint.config.mjs
   # - next-env.d.ts
   # - next.config.ts
   # - package-lock.json
   # - package.json
   # - postcss.config.mjs
   # - public/
   # - README.md
   # - tsconfig.json
   ```
   
   または、コマンドで削除する場合（PowerShell）:
   ```powershell
   cd frontend
   Remove-Item -Recurse -Force .next, app, node_modules, eslint.config.mjs, next-env.d.ts, next.config.ts, package-lock.json, package.json, postcss.config.mjs, public, README.md, tsconfig.json -ErrorAction SilentlyContinue
   ```

4. **Next.jsプロジェクトを作成（既存プロジェクトがない場合のみ）**
   ```bash
   npx create-next-app@latest . --typescript --tailwind --eslint --app --no-src-dir --import-alias "@/*" --yes
   ```

   **コマンドの説明**:
   - `npx`: パッケージを一時的に実行するコマンド
   - `create-next-app@latest`: Next.jsプロジェクトを作成するツール
   - `.`: 現在のフォルダ（frontend）に作成
   - `--typescript`: TypeScriptを使用（型安全なコードが書ける）
   - `--tailwind`: Tailwind CSSを使用（デザイン用）
   - `--eslint`: ESLintを使用（コードの品質チェック）
   - `--app`: App Routerを使用（Next.jsの新しい機能）
   - `--yes`: すべての質問に「はい」と答える

5. **インストールの完了を待つ（新規作成の場合）**
   - 数分かかります。`added 356 packages` のようなメッセージが表示されれば完了です。

## ✅ ステップ4: 環境変数の設定

### 環境変数とは？

**環境変数**: プログラムの設定値を保存するファイルです。パスワードやAPIキーなど、**秘密にすべき情報**を保存します。

**なぜ必要？**: 
- パスワードをコードに直接書くと、GitHubなどに公開したときに漏洩する危険があります
- 環境変数ファイル（`.env.local`）は `.gitignore` に含まれているため、Gitに追加されません

---

### 4-1. SupabaseのAPIキーを取得

**手順**:

1. **Supabase Dashboardを開く**
   - https://supabase.com/dashboard にアクセス
   - 作成したプロジェクトを選択

2. **Settings（設定）を開く**
   - 左側のメニューから「Settings」をクリック
   - 「API」をクリック

3. **APIキーを確認**
   - **Project URL**: このURLをコピー（例: `https://xxxxxxxxxxxxx.supabase.co`）
   - **anon public key**: このキーをコピー（`eyJ...` で始まる長い文字列）
   - **service_role key**: このキーもコピー（⚠️ **重要**: このキーは秘密にしてください！）

**各キーの説明**:

| キー | 説明 | 使用場所 |
|------|------|----------|
| **Project URL** | SupabaseプロジェクトのURL | フロントエンドとバックエンドの両方 |
| **anon public key** | 公開しても安全なキー（匿名キー） | フロントエンド（ブラウザ側） |
| **service_role key** | 管理者権限を持つキー（秘密） | バックエンド（サーバー側のみ） |

⚠️ **重要**: `service_role key` は絶対に公開しないでください！このキーはサーバー側でのみ使用します。

---

### 4-2. フロントエンドの環境変数ファイルを作成

**手順**:

1. **ファイルを作成**
   - `frontend` フォルダを開く
   - 新しいファイルを作成し、名前を `.env.local` にする
     - ⚠️ **重要**: ファイル名は `.env.local` です（先頭にドットがつきます）
     - Windowsでドットで始まるファイルを作る方法:
       - メモ帳を開く
       - 名前を付けて保存 → ファイル名を `.env.local` にする（拡張子なし）
       - 「すべてのファイル」を選択して保存

2. **以下の内容をコピー＆ペースト**
   ```env
   # Supabase（データベース・API設定）
   NEXT_PUBLIC_SUPABASE_URL=https://xxxxxxxxxxxxx.supabase.co
   NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
   
   # LINE Login（LINEログイン設定 - 後で設定）
   NEXT_PUBLIC_LINE_CHANNEL_ID=your_line_channel_id
   
   # Stripe（決済サービス設定 - 後で設定）
   NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY=your_stripe_publishable_key
   
   # NextAuth（認証ライブラリ設定 - 後で設定）
   NEXTAUTH_URL=http://localhost:3000
   NEXTAUTH_SECRET=your_nextauth_secret
   
   # アプリケーションURL
   NEXT_PUBLIC_APP_URL=http://localhost:3000
   ```

3. **Supabaseの値を設定**
   - `NEXT_PUBLIC_SUPABASE_URL`: 先ほどコピーした「Project URL」を貼り付け
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`: 先ほどコピーした「anon public key」を貼り付け

**各項目の説明**:

| 項目 | 説明 | 今すぐ設定が必要？ |
|------|------|-------------------|
| `NEXT_PUBLIC_SUPABASE_URL` | SupabaseプロジェクトのURL | ✅ **はい**（今すぐ設定） |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Supabaseの匿名キー | ✅ **はい**（今すぐ設定） |
| `NEXT_PUBLIC_LINE_CHANNEL_ID` | LINEログインの設定 | ❌ 後で設定（2/5以降） |
| `NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY` | 決済サービスの設定 | ❌ 後で設定（2/17以降） |
| `NEXTAUTH_URL` | 認証ライブラリのURL | ❌ そのままでOK |
| `NEXTAUTH_SECRET` | 認証ライブラリの秘密鍵 | ❌ 後で設定 |
| `NEXT_PUBLIC_APP_URL` | アプリケーションのURL | ❌ そのままでOK |

⚠️ **注意**: `NEXT_PUBLIC_` で始まる変数は、ブラウザ側（フロントエンド）に公開されます。機密情報には使用しないでください。

```
Either try using a new directory name, or remove the files listed above.
```

**原因**: `frontend`フォルダに既にNext.jsプロジェクトが存在しています。

**解決方法**:

**方法1: 既存プロジェクトを使用する（推奨）**
- 既にNext.jsプロジェクトが作成されている場合は、そのまま使用できます
- 次のステップ（Supabase Clientのインストール）に進んでください
- 既存プロジェクトの設定を確認:
  ```bash
  cd frontend
  cat package.json
  ```
  - `next`、`react`、`typescript`、`tailwindcss`が含まれていればOKです

**方法2: 既存プロジェクトを削除して再作成する**
- 既存のプロジェクトを削除したい場合のみ実行してください
- ⚠️ **注意**: 既存のコードが削除されます

  **Windows（PowerShell）の場合**:
  ```powershell
  cd frontend
  Remove-Item -Recurse -Force .next, app, node_modules, eslint.config.mjs, next-env.d.ts, next.config.ts, package-lock.json, package.json, postcss.config.mjs, public, README.md, tsconfig.json -ErrorAction SilentlyContinue
  ```
  
  その後、再度Next.jsプロジェクトを作成:
  ```bash
  npx create-next-app@latest . --typescript --tailwind --eslint --app --no-src-dir --import-alias "@/*" --yes
  ```

### エラー2: Supabase接続エラー

**エラーメッセージ例**:
```
Invalid API key
```

**原因**: SupabaseのAPIキーが間違っている、または環境変数が設定されていません。

**解決方法**:

1. **環境変数を確認**
   - `frontend/.env.local` の `NEXT_PUBLIC_SUPABASE_URL` と `NEXT_PUBLIC_SUPABASE_ANON_KEY` が正しく設定されているか確認
   - Supabase Dashboard > Settings > API から正しい値をコピー

2. **環境変数の形式を確認**
   - 値に余分なスペースや引用符がないか確認
   - 例: `NEXT_PUBLIC_SUPABASE_URL=https://xxxxxxxxxxxxx.supabase.co`（引用符なし）

3. **サーバーを再起動**
   - 環境変数を変更した場合は、フロントエンドサーバーを再起動してください

### エラー4: `.env.local` ファイルが見つからない

**エラーメッセージ例**:
```
NEXT_PUBLIC_SUPABASE_URL is not defined
```

**原因**: `.env.local` ファイルが作成されていない、または間違った場所にあります。

**解決方法**:

1. **ファイルの場所を確認**
   - `frontend/.env.local` は `frontend` フォルダの直下にあるか確認

2. **ファイル名を確認**
   - ファイル名は `.env.local` です（先頭にドットがつきます）
   - `env.local` や `.env.local.txt` では動作しません

3. **環境変数の確認**
   - `NEXT_PUBLIC_SUPABASE_URL` と `NEXT_PUBLIC_SUPABASE_ANON_KEY` が設定されているか確認

## 💡 よくある質問（FAQ）

### Q1: ターミナルとコマンドプロンプトは同じものですか？

**A**: ほぼ同じです。Windowsでは「コマンドプロンプト」や「PowerShell」を使用します。このガイドでは「ターミナル」と表記していますが、Windowsの場合はコマンドプロンプトやPowerShellを使用してください。

---

### Q2: `cd` コマンドとは何ですか？

**A**: `cd` は「Change Directory（ディレクトリを変更）」の略です。フォルダを移動するコマンドです。

- `cd frontend` → `frontend` フォルダに移動
- `cd ..` → 一つ上のフォルダに移動
- `cd C:\Users\tokam\Downloads\soccer` → 指定したフォルダに移動

---

### Q3: `npm install` と `npm run dev` の違いは？

**A**:
- `npm install`: パッケージ（ライブラリ）をインストールするコマンド
- `npm run dev`: 開発サーバーを起動するコマンド

---

### Q4: サーバーを起動したままにしても大丈夫ですか？

**A**: 開発中は起動したままでも問題ありません。ただし、作業が終わったら `Ctrl + C` で停止してください。

---

### Q5: エラーが出たときはどうすればいいですか？

**A**: 
1. エラーメッセージをよく読む
2. このガイドの「トラブルシューティング」を確認
3. それでも解決しない場合は、エラーメッセージをコピーして検索してみる

---
